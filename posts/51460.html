查看数据库

show databases;

选择数据库

use databasesName;

查看该数据库中有哪些表

show tables;

查询表中数据

select * from tableName;

退出数据库服务器

exit;

删除数据表

drop table tableName;

创建数据库

create database databaseName;

创建数据表

以创建student表为例：

create table student(
  id int primary key auto_increment,
  name VARCHAR(20),
  gender CHAR(1),
  birth DATE,
  death DATE );

注意：1.var()定常:哪怕存储的字符串没有达到”()”中数字的上限,var()依然会占用空格来填充空间.varchar()不定长,没有达到”()”中的上限则会自动去掉后面的空格;

2.定义最后一个字段的时候不要加”,”;

3.上面的”VAR”,”VARCHAR”,”DATE”可以用小写.不过最好用大写来表示区分关键字;

4.切换到英语状态下半角输入；

查看数据表表结构

desc tableName;

以student表为例：

mysql> desc student;
+--------+-------------+------+-----+---------+----------------+
| Field  | Type        | Null | Key | Default | Extra          |
+--------+-------------+------+-----+---------+----------------+
| id     | int         | NO   | PRI | NULL    | auto_increment |
| name   | varchar(20) | YES  |     | NULL    |                |
| gender | char(1)     | YES  |     | NULL    |                |
| birth  | date        | YES  |     | NULL    |                |
| death  | date        | YES  |     | NULL    |                |
+--------+-------------+------+-----+---------+----------------+
5 rows in set (0.01 sec)

说明：

Field ：字段名称

Type ：字段类型，比如 int , date , char , varchar

Key : 是否是关键字，比如可以定义为： primary key 或者 unique key

Default : 若是该字段没有主动设置值的时候,定义该字段的默认值

Extra : auto_increment 表示自增；

插入数据

insert into student values(null,'BingoLin','1','1995-01-01',null);

注意：是 null ，不是 ‘Null’

或者

insert into student(name,gender,birth) values('BingoLin','1','1995-01-01');

此时，已经向student增加了两条记录，这两条记录除了id 外一模一样。

使用select命令查看表，显示如下：

mysql> select * from student;
+----+----------+--------+------------+-------+
| id | name     | gender | birth      | death |
+----+----------+--------+------------+-------+
|  1 | BingoLin | 1      | 1995-01-01 | NULL  |
|  2 | BingoLin | 1      | 1995-01-01 | NULL  |
+----+----------+--------+------------+-------+
2 rows in set (0.00 sec)

注意： NULL:代表的是空,表示该字段还没有数据.

删除数据

delete from tableName where <条件>

以student 表为例:

delete from student where id = 2;

上述命令，意思是删除 student 表 中 id 为 2 的记录。

删除后的表查询结果如下：

mysql> select * from student;
+----+----------+--------+------------+-------+
| id | name     | gender | birth      | death |
+----+----------+--------+------------+-------+
|  1 | BingoLin | 1      | 1995-01-01 | NULL  |
+----+----------+--------+------------+-------+
1 row in set (0.00 sec)

修改数据

update tableName set 字段1=值1,字段2=值2 ... where <条件>;

以student 表为例:

update student 
set gender='0',
birth = '1995-02-01' 
where name = 'BingoLin' ;

以上例子表示： 修改student 表 中 name 字段值为 BingoLin 的记录。修改方案是将Bingo Lin所在的这条记录中 的gender字段值设置为0，birth字段值设置为1995-02-01。

修改完成后，表查询结果如下：

mysql> select * from student;
+----+----------+--------+------------+-------+
| id | name     | gender | birth      | death |
+----+----------+--------+------------+-------+
|  1 | BingoLin | 0      | 1995-02-01 | NULL  |
+----+----------+--------+------------+-------+
1 row in set (0.00 sec)

建表约束

1.主键约束

作用：唯一确定一张表中的一条记录。增加主键约束之后，使字段不重复且不为空；

以创建user表为例：

create table user(
id int primary key,
name varchar(20)
);

向user表添加记录：

insert into user values(1,'BingoLin');

添加记录后表查询结果如下：

mysql> select * from user;
+----+----------+
| id | name     |
+----+----------+
|  1 | BingoLin |
+----+----------+
1 row in set (0.00 sec)

表结构如下：

mysql> desc user;
+-------+-------------+------+-----+---------+-------+
| Field | Type        | Null | Key | Default | Extra |
+-------+-------------+------+-----+---------+-------+
| id    | int         | NO   | PRI | NULL    |       |
| name  | varchar(20) | YES  |     | NULL    |       |
+-------+-------------+------+-----+---------+-------+
2 rows in set (0.00 sec)

可以看到：id 的Null字段值为No,即 id 不为空；id 的 Key 字段值为 PRI,即（primary）

2.复合主键

以创建user2表为例：

create table user2(
id int,
name varchar(20),
password varchar(20),
primary key(id,name)
);

向user2添加记录；

insert into user2 values(1,'BingoLin','123456');
insert into user2 values(2,'BingoLi','123456');
insert into user2 values(3,'BingoLin','123456');

添加记录后表查询结果如下：

mysql> select * from user2;
+----+----------+----------+
| id | name     | password |
+----+----------+----------+
|  1 | BingoLin | 123456   |
|  2 | BingoLi  | 123456   |
|  3 | BingoLin | 123456   |
+----+----------+----------+
3 rows in set (0.00 sec)

表结构如下：

mysql> desc user2;
+----------+-------------+------+-----+---------+-------+
| Field    | Type        | Null | Key | Default | Extra |
+----------+-------------+------+-----+---------+-------+
| id       | int         | NO   | PRI | NULL    |       |
| name     | varchar(20) | NO   | PRI | NULL    |       |
| password | varchar(20) | YES  |     | NULL    |       |
+----------+-------------+------+-----+---------+-------+
3 rows in set (0.00 sec)

可以看到，添加 复合主键 的字段不为空且不重复。没有添加复合主键的字段值允许相同；

3.自增约束

以创建user3为例：

create table user3(
id int primary key auto_increment,
name varchar(20)
);

向user3添加记录

insert into user3(name) values ('BingoLin');
insert into user3(name) values ('BingoLi');

添加记录后表查询结果如下：

mysql> select * from user3;
+----+----------+
| id | name     |
+----+----------+
|  1 | BingoLin |
|  2 | BingoLi  |
+----+----------+
2 rows in set (0.00 sec)

表结构如下：

mysql> desc user3;
+-------+-------------+------+-----+---------+----------------+
| Field | Type        | Null | Key | Default | Extra          |
+-------+-------------+------+-----+---------+----------------+
| id    | int         | NO   | PRI | NULL    | auto_increment |
| name  | varchar(20) | YES  |     | NULL    |                |
+-------+-------------+------+-----+---------+----------------+
2 rows in set (0.00 sec)

说明：auto_increment 表示自增。可以看到 id 添加了auto_increment 属性后，即使没有自定义id 值，也能自动生成id

4.唯一约束

以创建user4为例：

create table user4(
id int primary key auto_increment,
name varchar(20) unique key
);

向user4 添加记录

insert into user4(name) values('BingoLin');
insert into user4(name) values('BingoLi');

添加记录后表结构查询如下：

mysql> select * from user4;
+----+----------+
| id | name     |
+----+----------+
|  2 | BingoLi  |
|  1 | BingoLin |
+----+----------+
2 rows in set (0.00 sec)

表结构如下：

mysql> desc user4;
+-------+-------------+------+-----+---------+----------------+
| Field | Type        | Null | Key | Default | Extra          |
+-------+-------------+------+-----+---------+----------------+
| id    | int         | NO   | PRI | NULL    | auto_increment |
| name  | varchar(20) | YES  | UNI | NULL    |                |
+-------+-------------+------+-----+---------+----------------+
2 rows in set (0.00 sec)

说明：可以看到，name 所在的 Key 的值是UNI 即Unique，

此时,如果还插入相同的name值的记录，比如：

insert into user4(name) values('BingoLi');

会出现错误提示：

mysql> insert into user4(name) values('BingoLi');
ERROR 1062 (23000): Duplicate entry 'BingoLi' for key 'user4.name'

注意：主键约束(primary key) 包含了唯一约束。

5.非空约束

以创建user5表为例：

create table user5(
id int primary key auto_increment,
name varchar(20) not null,
idCard varchar(20) unique key not null
);

向user5添加记录

insert into user5(name,idCard) values ('BingoLin','123456');
insert into user5(name,idCard) values ('BingoLi','33144245');

添加记录后表查询结果如下：

mysql> select * from user5;
+----+----------+----------+
| id | name     | idCard   |
+----+----------+----------+
|  1 | BingoLin | 123456   |
|  2 | BingoLi  | 33144245 |
+----+----------+----------+
2 rows in set (0.00 sec)

表结构如下：

mysql> desc user5;
+--------+-------------+------+-----+---------+----------------+
| Field  | Type        | Null | Key | Default | Extra          |
+--------+-------------+------+-----+---------+----------------+
| id     | int         | NO   | PRI | NULL    | auto_increment |
| name   | varchar(20) | NO   |     | NULL    |                |
| idCard | varchar(20) | NO   | UNI | NULL    |                |
+--------+-------------+------+-----+---------+----------------+
3 rows in set (0.00 sec)

说明：name,idCard 都设置了 not null ,即非空。此时，如果只添加含有name的记录：

insert into user5(name) values('Bingo');

则会出现错误：

mysql> insert into user5(name) values('Bingo');
ERROR 1364 (HY000): Field 'idCard' doesn't have a default value

只添加含有idCard 的记录也会出错；

结论：两个非空参数必须一起设置；

6.默认约束

以创建user6表为例：

create table user6(
id int primary key auto_increment,
name varchar(20) not null,
idCard varchar(20) unique key not null ,
status int default 0 comment'用户状态 0:启用 1:封禁 默认:0'
);

向user6添加记录：

insert into user6(name,idCard) values ('BingoLin','123456');
insert into user6(name,idCard) values ('BingoLi','1234567');

添加记录后表查询如下：

mysql> select * from user6;
+----+----------+---------+--------+
| id | name     | idCard  | status |
+----+----------+---------+--------+
|  1 | BingoLin | 123456  |      0 |
|  2 | BingoLi  | 1234567 |      0 |
+----+----------+---------+--------+
2 rows in set (0.00 sec)

说明：没有设置status的值，默认为0。

表结构如下：

mysql> desc user6;
+--------+-------------+------+-----+---------+----------------+
| Field  | Type        | Null | Key | Default | Extra          |
+--------+-------------+------+-----+---------+----------------+
| id     | int         | NO   | PRI | NULL    | auto_increment |
| name   | varchar(20) | NO   |     | NULL    |                |
| idCard | varchar(20) | NO   | UNI | NULL    |                |
| status | int         | YES  |     | 0       |                |
+--------+-------------+------+-----+---------+----------------+
4 rows in set (0.00 sec)

7.外键约束

以创建classes表、student表为例：

create table classes(
id int primary key auto_increment,
name varchar(20) unique not null 
);

向classes表中添加数据：

insert into classes(name) values('一班');
insert into classes(name) values('二班');
insert into classes(name) values('三班');
insert into classes(name) values('四班');

添加记录后表查询如下：

mysql> select * from classes;
+----+--------+
| id | name   |
+----+--------+
|  1 | 一班   |
|  2 | 二班   |
|  3 | 三班   |
|  4 | 四班   |
+----+--------+
4 rows in set (0.00 sec)

表结构如下：

mysql> desc classes;
+-------+-------------+------+-----+---------+----------------+
| Field | Type        | Null | Key | Default | Extra          |
+-------+-------------+------+-----+---------+----------------+
| id    | int         | NO   | PRI | NULL    | auto_increment |
| name  | varchar(20) | NO   | UNI | NULL    |                |
+-------+-------------+------+-----+---------+----------------+
2 rows in set (0.00 sec)

现在创建student表：

create table student(
id int primary key auto_increment,
name varchar(20) not null,
idCard varchar(20) unique key not null,
class_id int,
foreign key(class_id) references classes(id)
);

注：foreign key 外键；references 参考

向student表添加数据：

insert into student(name,idCard,class_id) values ('BingoLin',123456,1);
insert into student(name,idCard,class_id) values ('BingoLi',223456,2);
insert into student(name,idCard,class_id) values ('BingoL',323456,3);
insert into student(name,idCard,class_id) values ('Bingo',423456,4);

添加记录后表查询结果如下：

mysql> select * from student;
+----+----------+--------+----------+
| id | name     | idCard | class_id |
+----+----------+--------+----------+
|  1 | BingoLin | 123456 |        1 |
|  2 | BingoLi  | 223456 |        2 |
|  3 | BingoL   | 323456 |        3 |
|  4 | Bingo    | 423456 |        4 |
+----+----------+--------+----------+
4 rows in set (0.00 sec)

表结构如下：

mysql> desc student;
+----------+-------------+------+-----+---------+----------------+
| Field    | Type        | Null | Key | Default | Extra          |
+----------+-------------+------+-----+---------+----------------+
| id       | int         | NO   | PRI | NULL    | auto_increment |
| name     | varchar(20) | NO   |     | NULL    |                |
| idCard   | varchar(20) | NO   | UNI | NULL    |                |
| class_id | int         | YES  | MUL | NULL    |                |
+----------+-------------+------+-----+---------+----------------+
4 rows in set (0.00 sec)

说明：因为存在外键，如果想在student表添加班级为5的数据，是行不通的，因为classes表还没有五班的数据：

mysql> insert into student(name,idCard,class_id) values ('BingoLin',523456,5);
ERROR 1452 (23000): Cannot add or update a child row: a foreign key constraint fails (`school`.`student`, CONSTRAINT `student_ibfk_1` FOREIGN KEY (`class_id`) REFERENCES `classes` (`id`))

并且，因为student表中属于一班的同学是存在的，那么将无法删除classes表中一班的数据：

mysql> delete from classes where name = '一班';
ERROR 1451 (23000): Cannot delete or update a parent row: a foreign key constraint fails (`school`.`student`, CONSTRAINT `student_ibfk_1` FOREIGN KEY (`class_id`) REFERENCES `classes` (`id`))

办法：只有先删除掉student表中关于一班的所有数据，才能在classes表中删除一班的数据。

8.建表之后再添加/删除主键约束

以创建user8为例：

create table user8(
id int,
name varchar(20)
);

表结构如下：

mysql> desc user8;
+-------+-------------+------+-----+---------+-------+
| Field | Type        | Null | Key | Default | Extra |
+-------+-------------+------+-----+---------+-------+
| id    | int         | YES  |     | NULL    |       |
| name  | varchar(20) | YES  |     | NULL    |       |
+-------+-------------+------+-----+---------+-------+
2 rows in set (0.00 sec)

为user8表id 添加主键约束：

alter table user8 add primary key(id);

添加主键约束后表结构如下：

mysql> desc user8;
+-------+-------------+------+-----+---------+-------+
| Field | Type        | Null | Key | Default | Extra |
+-------+-------------+------+-----+---------+-------+
| id    | int         | NO   | PRI | NULL    |       |
| name  | varchar(20) | YES  |     | NULL    |       |
+-------+-------------+------+-----+---------+-------+
2 rows in set (0.00 sec)

将user8表id 删除主键约束：

alter table user8 drop primary key;

删除主键约束后表结构如下：

mysql> desc user8;
+-------+-------------+------+-----+---------+-------+
| Field | Type        | Null | Key | Default | Extra |
+-------+-------------+------+-----+---------+-------+
| id    | int         | NO   |     | NULL    |       |
| name  | varchar(20) | YES  |     | NULL    |       |
+-------+-------------+------+-----+---------+-------+
2 rows in set (0.00 sec)

也可以再使用modify 修改字段从而添加主键约束：

alter table user8 modify id int primary key;

修改后查看表结构：

mysql> desc user8;
+-------+-------------+------+-----+---------+-------+
| Field | Type        | Null | Key | Default | Extra |
+-------+-------------+------+-----+---------+-------+
| id    | int         | NO   | PRI | NULL    |       |
| name  | varchar(20) | YES  |     | NULL    |       |
+-------+-------------+------+-----+---------+-------+
2 rows in set (0.00 sec)

给主键设置自增：

alter table user8 modify id int  auto_increment;

给主键设置自增后表结构：

mysql> desc user8;
+-------+-------------+------+-----+---------+----------------+
| Field | Type        | Null | Key | Default | Extra          |
+-------+-------------+------+-----+---------+----------------+
| id    | int         | NO   | PRI | NULL    | auto_increment |
| name  | varchar(20) | YES  |     | NULL    |                |
+-------+-------------+------+-----+---------+----------------+
2 rows in set (0.01 sec)
